1 Differentiate between MFT & MVT
Answer

MFT vs MVT - Memory Management Techniques

MFT (Multiprogramming with Fixed Tasks)

  • Memory Division: Memory is divided into fixed-size partitions
  • Partition Size: Each partition has predetermined size
  • Job Allocation: Jobs are allocated to partitions that can accommodate them
  • Internal Fragmentation: High internal fragmentation due to fixed sizes
  • Memory Utilization: Lower memory utilization
  • Implementation: Simpler to implement and manage
  • Overhead: Less overhead in memory management

MVT (Multiprogramming with Variable Tasks)

  • Memory Division: Memory is divided into variable-size partitions
  • Partition Size: Partitions are created dynamically based on job requirements
  • Job Allocation: Jobs are allocated exactly the memory they need
  • Internal Fragmentation: No internal fragmentation
  • Memory Utilization: Higher memory utilization
  • Implementation: More complex to implement
  • Overhead: Higher overhead due to dynamic allocation
Key Differences Explained:
Memory Allocation:

MFT uses fixed partitions while MVT uses dynamic partitions that adjust to job sizes.

Fragmentation:

MFT suffers from internal fragmentation, MVT may have external fragmentation.

Efficiency:

MVT provides better memory utilization but requires more complex management.

Complexity:

MFT is simpler to implement, MVT requires sophisticated memory management algorithms.

Swapping in Operating Systems

Swapping is a memory management technique where processes are moved between main memory and secondary storage (disk) to allow more processes to run than can fit in physical memory.

  • Advantages of Swapping:
    • Allows more processes to run than physical memory can hold
    • Increases degree of multiprogramming
    • Better CPU utilization
    • Enables running large programs on limited memory
  • Disadvantages of Swapping:
    • High overhead due to disk I/O operations
    • Context switching time increases
    • Requires sufficient disk space for swap area
    • Can cause thrashing if overused
  • When Swapping Occurs:
    • When a higher priority process needs memory
    • When a process has been idle for a long time
    • When system runs out of physical memory
    • During memory compaction operations
6 Describe variable partitioning technique of memory management
Answer

Variable Partitioning Memory Management

Variable partitioning is a dynamic memory allocation technique where memory is divided into variable-sized partitions based on the actual size requirements of processes.

Variable Partitioning Example
Process A (30KB)
Free (10KB)
Process B (25KB)
Process C (20KB)
Free (15KB)
  • How Variable Partitioning Works:
    • Memory is initially one large free block
    • When a process arrives, it's allocated exactly the memory it needs
    • Partitions are created dynamically based on process size
    • When a process terminates, its memory becomes available
  • Memory Allocation Strategies:
    • First Fit: Allocate first available block that fits
    • Best Fit: Allocate smallest available block that fits
    • Worst Fit: Allocate largest available block
    • Next Fit: Start searching from last allocation point
  • Advantages:
    • No internal fragmentation
    • Better memory utilization than fixed partitioning
    • Flexible allocation based on actual needs
    • Can accommodate processes of varying sizes
  • Disadvantages:
    • External fragmentation occurs over time
    • Complex memory management required
    • Memory compaction may be needed
    • Higher overhead for allocation/deallocation
7 Describe necessary condition for deadlock
Answer

Necessary Conditions for Deadlock

Deadlock occurs when two or more processes are blocked forever, waiting for each other to release resources. Four necessary conditions must be present simultaneously for a deadlock to occur.

1. Mutual Exclusion

At least one resource must be held in a non-sharable mode. Only one process can use the resource at a time.

Example: Printer, tape drive, or any exclusive resource
2. Hold and Wait

A process must be holding at least one resource and waiting to acquire additional resources held by other processes.

Example: Process A holds printer, waits for scanner held by Process B
3. No Preemption

Resources cannot be preempted. A resource can only be released voluntarily by the process holding it.

Example: Cannot force a process to release a printer it's using
4. Circular Wait

There must exist a circular chain of processes, each waiting for a resource held by the next process in the chain.

Example: P1 waits for R2, P2 waits for R3, P3 waits for R1
Deadlock Example
Process A
Resource 1
Process B
Resource 2
Holds
Waits for
Holds
Waits for

Result: Both processes are blocked, waiting for each other's resources.

Deadlock Prevention Strategies
  • Eliminate Mutual Exclusion: Make resources shareable (not always possible)
  • Eliminate Hold and Wait: Require processes to request all resources at once
  • Allow Preemption: Force processes to release resources when needed
  • Eliminate Circular Wait: Impose ordering on resource requests
8 Explain Bankers Algorithm to avoid Deadlock
Answer

Banker's Algorithm for Deadlock Avoidance

The Banker's Algorithm is a deadlock avoidance algorithm that ensures the system never enters an unsafe state. It's named after banking systems where customers request loans.

Algorithm Components
Available Resources

Vector showing available instances of each resource type

Maximum Need

Maximum resources each process may need

Allocation

Resources currently allocated to each process

Need

Remaining resources needed by each process (Max - Allocation)

Safety Algorithm Steps
1
Initialize

Work = Available, Finish[i] = false for all processes

2
Find Process

Find process i where Finish[i] = false and Need[i] ≤ Work

3
Update

Work = Work + Allocation[i], Finish[i] = true

4
Check

If all processes finished, system is safe

Banker's Algorithm Example
System State:
Process Allocation Max Need Available
P0 0 1 0 7 5 3 7 4 3 3 3 2
P1 2 0 0 3 2 2 1 2 2
P2 3 0 2 9 0 2 6 0 0
P3 2 1 1 2 2 2 0 1 1
  • Resource Request Algorithm:
    • Check if request ≤ need for the process
    • Check if request ≤ available resources
    • Tentatively allocate resources
    • Run safety algorithm to check if state is safe
    • If safe, grant request; otherwise, deny and restore state
  • Advantages:
    • Prevents deadlock by avoiding unsafe states
    • Allows maximum resource utilization
    • Works with multiple resource types
  • Disadvantages:
    • Requires advance knowledge of maximum resource needs
    • High computational overhead
    • Number of processes must be fixed

File blocks are allocated in consecutive disk blocks. Simple but suffers from external fragmentation.

File blocks are linked together using pointers. No external fragmentation but poor random access.

3 Define virtual memory & paging
Answer

Virtual Memory and Paging

Virtual Memory

A memory management technique that allows programs to use more memory than physically available by using secondary storage as an extension of main memory.

  • Creates illusion of large memory space
  • Allows programs larger than physical memory
  • Enables multiprogramming with limited RAM
Paging

A memory management scheme where physical memory is divided into fixed-size blocks called frames, and logical memory is divided into same-size blocks called pages.

  • Eliminates external fragmentation
  • Allows non-contiguous allocation
  • Uses page table for address translation
4 Define Segmentation & compaction & Fragmentation
Answer

Segmentation, Compaction, and Fragmentation

Segmentation

Memory management technique where logical address space is divided into variable-sized segments based on program structure (code, data, stack).

  • Segments have logical meaning
  • Variable size segments
  • Uses segment table for translation
Compaction

Process of moving allocated memory blocks to eliminate external fragmentation by placing all free memory in one contiguous block.

  • Reduces external fragmentation
  • Requires process relocation
  • Expensive operation
Fragmentation

Waste of memory space due to inability to use available memory blocks effectively.

  • Internal: Unused space within allocated blocks
  • External: Unused space between allocated blocks
5 Features of non Pre-emptive
Answer

Features of Non-Preemptive Scheduling

Process Control

Once a process starts executing, it continues until completion or voluntary release of CPU.

No Interruption

Running process cannot be interrupted by higher priority processes or time quantum expiration.

Context Switching

Context switching occurs only when process terminates or blocks for I/O operation.

Scheduling Decision

Scheduling decisions are made only when CPU becomes idle or process blocks.

Fairness

Ensures each process gets complete CPU time but may cause starvation of short processes.

Implementation

Simpler to implement as no need to handle preemption and process state saving.

6 Define Deadlock
Answer

Deadlock Definition

Deadlock is a situation in a multi-process system where two or more processes are blocked forever, waiting for each other to release resources that they need to continue execution.

Key Characteristics:
  • Circular Wait: Processes form a circular chain of resource requests
  • Resource Hold: Each process holds at least one resource
  • Mutual Exclusion: Resources cannot be shared simultaneously
  • No Preemption: Resources cannot be forcibly taken away
Simple Example:

Process A holds Resource 1 and waits for Resource 2
Process B holds Resource 2 and waits for Resource 1
Result: Both processes are stuck forever

8 State any four criteria in CPU scheduling
Answer

CPU Scheduling Criteria

CPU Utilization

Percentage of time CPU is busy executing processes. Higher utilization indicates better system performance.

Throughput

Number of processes completed per unit time. Higher throughput means more work done.

Turnaround Time

Total time from process submission to completion. Includes waiting time and execution time.

Waiting Time

Total time process spends waiting in ready queue. Should be minimized for better performance.

Response Time

Time from submission to first response. Important for interactive systems.

Fairness

Equal treatment of all processes. No process should be starved of CPU time.

9 Compare SJF & SRTN
Answer

SJF vs SRTN Comparison

SJF (Shortest Job First)

  • Type: Non-preemptive scheduling
  • Selection: Process with shortest burst time
  • Execution: Runs to completion
  • Advantage: Minimum average waiting time
  • Disadvantage: May cause starvation
  • Use Case: Batch processing systems

SRTN (Shortest Remaining Time Next)

  • Type: Preemptive scheduling
  • Selection: Process with shortest remaining time
  • Execution: Can be interrupted
  • Advantage: Better response time
  • Disadvantage: Higher overhead
  • Use Case: Interactive systems
Key Differences:
  • Preemption: SJF is non-preemptive, SRTN is preemptive
  • Decision Point: SJF decides at arrival, SRTN at every time quantum
  • Overhead: SRTN has higher context switching overhead
  • Starvation: Both can cause starvation of long processes
