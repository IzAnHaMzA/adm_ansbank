1 Differentiate between SQL and NoSQL database
Answer

SQL vs NoSQL Databases - Comprehensive Comparison

  • SQL Databases (Relational):
    • Structure: Tabular format with rows and columns, follows relational model
    • Schema: Fixed, predefined structure requiring all data to conform
    • ACID Properties: Atomicity, Consistency, Isolation, Durability fully guaranteed
    • Scalability: Vertical scaling (adding more powerful hardware)
    • Query Language: SQL (Structured Query Language) - standardized
    • Relationships: Strong support via foreign keys, joins, and referential integrity
    • Data Consistency: Strong consistency model
    • Use Cases: Financial systems, ERP, traditional business applications
    • Examples: MySQL, PostgreSQL, Oracle, SQL Server, SQLite
  • NoSQL Databases (Non-Relational):
    • Structure: Flexible models - document, key-value, column-family, graph
    • Schema: Schema-less or flexible, varying structures within same collection
    • ACID Properties: BASE model (Basically Available, Soft state, Eventual consistency)
    • Scalability: Excellent horizontal scaling across multiple servers
    • Query Language: Database-specific (MongoDB Query Language, CQL, etc.)
    • Relationships: Limited support, focuses on denormalization for performance
    • Data Consistency: Eventual consistency model
    • Use Cases: Big data, real-time apps, content management, IoT
    • Examples: MongoDB (Document), Cassandra (Column), Redis (Key-Value), Neo4j (Graph)
  • Key Differences:
    • Performance: SQL optimized for complex queries; NoSQL for simple queries and high throughput
    • Development Speed: SQL requires careful schema design; NoSQL allows rapid development
    • Data Volume: SQL handles structured data well; NoSQL excels with unstructured datasets
    • Team Skills: SQL has standardized skills; NoSQL requires database-specific knowledge
Key Differences Explained:
Data Model:

SQL uses relational model with tables, while NoSQL uses various models like documents, key-value pairs, columns, or graphs.

Consistency:

SQL prioritizes consistency (ACID), while NoSQL often prioritizes availability and partition tolerance (CAP theorem).

Performance:

SQL optimized for complex queries and transactions. NoSQL optimized for simple queries and high throughput.

Development:

SQL requires careful schema design upfront. NoSQL allows rapid development with flexible schema.

Explain 4 types of NoSQL database with diagram

Answer

4 Types of NoSQL Databases

  • 1. Document Databases:
    • Storage Format: Data stored as documents (JSON, BSON, XML)
    • Schema: Flexible schema - documents can have different structures
    • Use Cases: Content management, user profiles, product catalogs
    • Query Capabilities: Rich query language with complex queries and aggregation
    • Examples: MongoDB, CouchDB, Amazon DocumentDB, Azure Cosmos DB
  • 2. Column-Family Databases:
    • Storage Format: Data organized in column families (groups of related columns)
    • Schema: Sparse schema - not all rows need all columns
    • Use Cases: Time-series data, IoT applications, analytics
    • Query Capabilities: Fast reads and writes, excellent for analytics
    • Examples: Cassandra, HBase, Amazon DynamoDB, Google Bigtable
  • 3. Key-Value Databases:
    • Storage Format: Simple key-value pairs for data storage
    • Schema: No schema - values can be any data type
    • Use Cases: Caching, session storage, real-time recommendations
    • Query Capabilities: Very fast read/write operations
    • Examples: Redis, Amazon DynamoDB, Riak, Oracle NoSQL Database
  • 4. Graph Databases:
    • Storage Format: Data stored as nodes (entities) and edges (relationships)
    • Schema: Flexible schema - nodes and edges can have properties
    • Use Cases: Social networks, fraud detection, recommendation engines
    • Query Capabilities: Graph traversal queries, pattern matching
    • Examples: Neo4j, Amazon Neptune, ArangoDB, OrientDB

What is meant by querying in MongoDB (find() function)

Answer

MongoDB Querying - find() Function

Querying in MongoDB is the process of retrieving data from collections using various methods, primarily the find() function. MongoDB uses a document-oriented query language that provides flexible and powerful data retrieval capabilities.

  • Core Concepts:
    • Collections: Similar to tables in SQL but with flexible document structures
    • Documents: BSON (Binary JSON) objects representing data records
    • Queries: JavaScript-like objects that specify search criteria
    • Projections: Specify which fields to return in results
    • Indexes: Improve query performance through optimized data structures
    • Cursors: Pointers to result sets that can be iterated
  • find() Function Syntax:
    • Basic syntax: db.collection.find(query, projection, options)
    • Query: Document specifying selection criteria (optional, empty {} for all documents)
    • Projection: Document specifying fields to return (optional, 1 to include, 0 to exclude)
    • Options: Document specifying query options like limit, sort, skip (optional)
  • Common Examples:
    • Find all documents: db.users.find()
    • Find with filter: db.users.find({"age": 25})
    • Find with projection: db.users.find({}, {"name": 1, "email": 1})
    • Find with operators: db.users.find({"age": {"$gt": 18}})
    • Find with logical operators: db.users.find({"$or": [{"age": {"$gt": 30}}, {"city": "New York"}]})
    • Find with array operations: db.users.find({"skills": {"$in": ["JavaScript", "Python"]}})
    • Find with embedded documents: db.users.find({"address.city": "New York"})
    • Find with sorting and limiting: db.users.find().sort({"age": -1}).limit(5)
  • Query Operators:
    • Comparison: $eq, $gt, $gte, $lt, $lte, $ne
    • Logical: $and, $or, $not, $nor
    • Array: $in, $nin, $all, $size
    • Element: $exists, $type
Common Query Operators:
Comparison Operators:
  • $eq - Equal to
  • $gt - Greater than
  • $gte - Greater than or equal
  • $lt - Less than
  • $lte - Less than or equal
  • $ne - Not equal to
Logical Operators:
  • $and - Logical AND
  • $or - Logical OR
  • $not - Logical NOT
  • $nor - Logical NOR
Array Operators:
  • $in - Matches any value in array
  • $nin - Matches no value in array
  • $all - Matches all values in array
  • $size - Matches array size

Q7 4 Marks

What is XML Schema. Illustrate it with an example

XML Schema

XML Schema is a language for describing the structure and constraints of XML documents. It provides a way to define elements, attributes, data types, and relationships.

Example XML Schema:
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="book"> <xs:complexType> <xs:sequence> <xs:element name="title" type="xs:string"/> <xs:element name="author" type="xs:string"/> <xs:element name="price" type="xs:decimal"/> <xs:element name="year" type="xs:integer"/> </xs:sequence> <xs:attribute name="id" type="xs:integer" use="required"/> </xs:complexType> </xs:element> </xs:schema>
Corresponding XML Document:
<book id="1"> <title>Database Systems</title> <author>John Smith</author> <price>45.99</price> <year>2023</year> </book>

State difference between structured and unstructured data

Structured Data

  • Format: Fixed schema, tabular format
  • Storage: Relational databases, spreadsheets
  • Examples: Customer records, transaction data
  • Processing: Easy to query and analyze
  • Tools: SQL, Excel, traditional BI tools
  • Volume: Typically smaller datasets

Unstructured Data

  • Format: No fixed schema, various formats
  • Storage: NoSQL databases, file systems
  • Examples: Text documents, images, videos
  • Processing: Requires specialized tools
  • Tools: NLP, machine learning, Hadoop
  • Volume: Typically larger datasets

Q2 2 Marks

Difference between XPath and XQuery (any 2)

XPath

  • Purpose: Path language for selecting nodes in XML documents
  • Functionality: Navigate and locate elements in XML
  • Usage: Used within XSLT, XQuery, and other XML technologies
  • Syntax: Uses path expressions with axes and predicates

XQuery

  • Purpose: Query language for XML data sources
  • Functionality: Query, transform, and manipulate XML data
  • Usage: Standalone query language for XML databases
  • Syntax: FLWOR expressions (For, Let, Where, Order by, Return)

Explain basic data types and arrays in MongoDB

Answer

MongoDB Basic Data Types and Arrays

  • Basic Data Types:
    • String: UTF-8 encoded text data for names, descriptions
    • Integer: 32-bit or 64-bit whole numbers
    • Double: Floating-point decimal numbers
    • Boolean: true/false values for logical operations
    • Date: Date and time objects with built-in manipulation
  • Arrays:
    • Ordered collections of values in any BSON data type
    • Can store mixed data types in same array (unlike SQL)
    • Support nested arrays and array query operators ($in, $all, $size)
    • Automatically indexed with multikey indexes

for $book in doc("books.xml")/books/book where $book/year > 2020 and $book/price < 50 return $book/title

Q7 2 Marks

Describe FLWOR Expression

FLWOR Expression

FLWOR is an acronym for the main clauses in XQuery expressions that provide powerful querying capabilities.

For

Iterates over a sequence of items

for $x in (1, 2, 3)
Let

Binds a variable to a value

let $price := 100
Where

Filters the results based on conditions

where $price > 50
Order by

Sorts the results

order by $price descending
Return

Specifies what to return

return $price
